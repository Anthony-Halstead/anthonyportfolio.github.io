MeshForge ‚Äì Modular Character Editor

Unity Tools

üéØ Project Overview

MeshForge is a Unity editor toolkit for turning existing skinned and static meshes into modular, reusable building blocks. It‚Äôs aimed at developers and technical artists who don‚Äôt live in DCC tools all day, but still need flexible character and environment modularity. The suite bundles several tightly integrated tools‚ÄîMesh Rebaker, Splitter, Gap Filler, Skinned Mesh Combiner, and the newer Topology Splitter‚Äîinto a non-destructive workflow that runs entirely inside the Unity Editor.

The asset is compatible with Unity 2021 LTS through Unity 6, supports all major render pipelines, and has been iteratively shipped and updated on the Unity Asset Store (v1.2.0 as of April 2025).

üß© The Problem

Most small teams and solo devs don‚Äôt have time to constantly go back into Blender/Maya just to test a new armor variant, swap a character‚Äôs head, or break a static mesh into modular kit pieces. I built MeshForge to answer:

‚ÄúHow can Unity developers quickly modularize skinned and static meshes‚Äîwithout deep 3D modeling skills‚Äîwhile preserving rigging, materials, and animation fidelity, and keeping the workflow safe and non-destructive?‚Äù

Key challenges:

  • Preserving skin weights, bone influences, and animation behavior when splitting skinned meshes
  • Handling meshes with multiple submeshes and materials without breaking existing setups
  • Supporting both skinned and static meshes in a single, coherent workflow
  • Filling open gaps cleanly while keeping UVs and materials consistent
  • Keeping everything non-destructive so artists never have to re-import ‚Äúruined‚Äù assets
  • Designing algorithms that can later be migrated to Jobs/Burst for future runtime use
  • Maintaining compatibility across multiple Unity versions and render pipelines

üõ†Ô∏è What I Built

  • Mesh Rebaker ‚Äì Rebuilds meshes into Unity‚Äôs coordinate space from a prefab reference, giving cleaner, predictable geometry as a foundation for further edits.
  • Splitter ‚Äì Core tool that splits both skinned and static meshes into modular parts. For skinned meshes it:
    • Extracts each logical part into its own skinned mesh while preserving skin weights and animation behavior
    • Supports submesh extraction so material assignments remain intact
    • Includes experimental deformation options to resize parts to match different character scales (e.g., small ‚Üí large rigs)
  • Gap Filler ‚Äì Fills holes left after splitting by selecting boundary vertices and procedurally generating new triangles. Supports ‚Äúfan‚Äù (flat) and convex fill modes, reusing the original material or a user-specified override.
  • Skinned Mesh Combiner ‚Äì Recombines previously split parts back into a single skinned mesh. Useful for baking down finalized variants or reducing draw calls after experimentation.
  • Topology Splitter (v1.2.0) ‚Äì A topology-aware static mesh splitter that analyzes connected components and lets users group, merge, and cap specific parts. This is especially useful for environment meshes and intricate props.
  • Non-destructive workflow ‚Äì All operations work on instances or duplicated assets, keeping the original files intact so users can experiment freely without re-importing content.
  • Editor-focused UX ‚Äì Contextual inspector UI, drag-select support, and clear feedback for long-running operations; plus documentation, tutorial videos, and a Discord community for support.

üé® Technical & Design Principles Applied

  • Topology-driven operations ‚Äì Many features are built around understanding mesh connectivity so that parts can be separated or grouped in predictable, artist-friendly ways.
  • Data-preserving edits ‚Äì Careful handling of vertex indices, bone weights, bindposes, and UVs to maintain animation fidelity and texture alignment after splitting or filling.
  • Non-destructive editor tooling ‚Äì All tools are designed so they never overwrite the source asset, encouraging experimentation and iteration without fear of losing work.
  • Performance-aware architecture ‚Äì Core algorithms are structured to be portable to Unity‚Äôs C# Job System and Burst compiler so that future runtime variants can scale to larger scenes and more complex meshes.
  • Robust version & pipeline support ‚Äì Targeting Unity 2021 LTS‚ÄìUnity 6 with attention to editor API changes (e.g., drag-select behavior differences in pre-2022 versions) and compatibility with Built-in, URP, and HDRP.

‚úÖ Outcome

  • Shipped MeshForge as a production-ready Unity Editor extension on the Unity Asset Store, iterating up to v1.2.0 with new features like Topology Splitter and improved static mesh support.
  • Enabled developers to convert existing character and environment assets into modular kits without needing to open a DCC tool, improving iteration speed for prototyping, customization, and content variation.
